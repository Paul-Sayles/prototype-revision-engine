Add project brief**PROJECT BRIEF — Revision Machine (Trust-Owned, Reusable)**

I need to build a simple, browser-based revision question system for students (similar in feel to Sparx, but much smaller in scope).

**Purpose:**
Students should be able to log on (in or out of lessons), choose a topic, and work through exam-style questions to reinforce knowledge independently.

**Core Requirements (Version 1):**

* Runs in a browser (no installs)
* Topic-based navigation (e.g. OCR Engineering J823)
* One question at a time (like a slide)
* Multiple choice questions (MCQ) to start
* Immediate feedback (correct/incorrect + explanation)
* “Next question” flow (like PowerPoint but dynamic)
* End-of-set summary (score)

**Content Structure:**

* Questions stored externally (JSON or similar)
* Easy to add/modify questions without changing code
* Designed so other subjects can reuse the same engine

**Non-Goals (for now):**

* No full login system
* No advanced adaptive AI
* No analytics dashboards
* No full LMS features

**Nice-to-Have (if simple):**

* Basic progress tracking within a session
* Randomised question order
* Simple “weak topic repeat” behaviour

**Constraints:**

* Limited time
* Minimal coding experience
* Needs to be usable and demo-ready quickly
* Should be scalable across subjects if adopted by the trust

**Goal:**
Deliver a working prototype that:

* functions like a revision “slide engine”
* can be demonstrated to leadership
* can realistically be expanded later into a broader system

I want step-by-step guidance, minimal complexity, and working outputs at each stage.
